Study Background

This dashboard visualizes OSHA Severe Injury Report (SIR) data across three datasets: Equipment/Machinery injuries in construction (N=1,576), Hand/Upper Extremity injuries in construction (N=3,282), and Mining/Natural Resources injuries (N=2,464). Records span 2015 to 2023 and are coded using the Occupational Injury and Illness Classification System (OIICS) and North American Industry Classification System (NAICS). This tool is part of a 10-paper programmatic PhD research series at NC State University.

Marginal Probability Formula

P(Category) =
Count(Category)
─────────────────── × 100
Total Injuries (N)

95% Confidence intervals are calculated using the Goodman (1965) simultaneous multinomial method, which controls the family-wise error rate across all categories of a variable.

Conditional Probability Formula

P(A | B=b) =
Count(A ∩ B=b)
─────────────────── × 100
Count(B=b)

For example: P(Event="Falls" | Body="Head") gives the likelihood of a fall event given that the injured body part was the head. CIs follow Goodman (1965) conditional on the given subset.

Data Scope & Limitations

  • Federal-jurisdiction states only (mixed federal/state-plan coverage)
  • No exposure denominators — composition analysis, not incidence rates
  • Hospitalization: 1 = Hospitalized, 2 = Not hospitalized
  • Amputation: 1 = Amputation occurred, 2 = No amputation
  • Source: OSHA Severe Injury Reports public database

Pros & Cons: SIR Dashboard vs. Traditional Shiny Approach

Advantages of This Approach

  • No R/Shiny server required — runs entirely in the browser
  • All three datasets switchable in one interface (vs. separate apps)
  • PTP/JHA integration panel directly links injury stats to safety planning tools
  • Goodman CIs computed pre-analysis for fast rendering
  • Portable: single HTML file, works offline after load
  • Dark industrial aesthetic appropriate for safety/engineering audience
  • Chart.js provides interactive tooltips and hover states

Limitations vs. Full Shiny App

  • No server-side computation — all data pre-aggregated (raw record filtering not available)
  • Two-variable conditioning (Tab 4 equivalent) not implemented here
  • No date-range slider or geographic filtering
  • Static dataset: requires rebuild if new SIR records added
  • Chart.js horizontal bar CIs are approximate visual ranges, not true error bars
